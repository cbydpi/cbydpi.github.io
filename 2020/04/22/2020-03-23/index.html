2020-03-23——面试题整理

一、理解原型

​ 在js中都是对象，既然是对象，都会有属性和方法，但是每一个对象都在内存中创建属性和方法，其中肯定会有许多重复的，那怎么把这些重复的，可共用的属性和方法共享出来呢，就有了prototype(原型)属性，原型又包含了一个原型对象，那所有需要共享的属性和方法就可以放在里面了，当创建了一个实例，所有的实例对象共享了同一个原型对象，就形成了原型链，也实现了继承的目的。

二、call和apply的作用和区别

​ call和apply都是用来调用函数的，其作用也相同，改变this的指向，传递的第一个参数用来替换this指，call后面的参数是枚举传递，以逗号隔开，apply后面的参数是以数组的形式传递。

三、深、浅克隆

​ 浅克隆只是增加了一个指针指向被克隆的对象。

​ 实现方法：

​ 直接赋值;Object.assign()

​ 深克隆不仅增加了一个指针，还在内存中开辟了一块新的区域并指向它，与原有数据完全断开了联系。

​ 实现方法：

  • json序列化，适用于比较简单的对象的克隆;
  • 递归遍历，在内存中开辟新地址再赋值过来，需要注意各种值的类型；
  • 第三方库：lodash,jquery。

四、异步解决方案

  • es5：回调函数
  • promise：三种状态，pending,resolved,rejected,
    链式调用，每次返回都是一个新的promise,其中，then是异步执行的，内部代码是立即执行的，解决了回调地狱的问题，但是无法取消，需要用回调函数来捕获错误，是微任务。
  • async/await：号称异步的终极解决方案，基于promise,也是非阻塞式的，以看起来同步的方式来调用异步函数，和generate一样。

五、事件循环机制

​ 事件循环如字面意思一样，就是一个循环，

  • 主线程每次执行时，先看看要执行的是同步任务，还是异步的API

  • 同步任务就继续执行，一直执行完

  • 遇到异步API就将它交给对应的异步线程，自己继续执行同步任务

  • 异步线程执行异步API，执行完后，将异步回调事件放入事件队列上

  • 主线程手上的同步任务干完后就来事件队列看看有没有任务

  • 主线程发现事件队列有任务，就取出里面的任务执行

  • 主线程不断循环上述流程

​ 详细些还可分为宏任务和微任务：

  1. 一个Event Loop可以有一个或多个事件队列，但是只有一个微任务队列。
  2. 微任务队列全部执行完会重新渲染一次
  3. 每个宏任务执行完都会重新渲染一次
  4. requestAnimationFrame处于渲染阶段，不在微任务队列，也不在宏任务队列

​ 常见的宏任务有：

  1. script (可以理解为外层同步代码)

  2. setTimeout/setInterval

  3. setImmediate(Node.js)

  4. I/O

  5. UI事件

  6. postMessage

    常见的微任务有：

  7. Promise

  8. process.nextTick(Node.js)

  9. Object.observe

  10. MutaionObserver

七、MVVM

​ Model-View-ViewModel,将其中的view的状态和行为抽象化，让我们可以将UI和业务逻辑分开，其优点是低耦合、可重用性、独立开发。

  • View 接收用户交互请求
  • View 将请求转交给ViewModel
  • ViewModel 操作Model数据更新
  • Model 更新完数据，通知ViewModel数据发生变化
  • ViewModel 更新View数据

八、双向绑定

​ vue2通过object.defineProperty支持data属性的setter,getter，在数据变动时发布消息给订阅者，触发相应的监听回调。

​ 缺点：无法监听数组的变化，虽然VUE中列举出了一些可以监听数据变化的方法，但是总的来说支持度还是不太高的。可以使用watch来进行深度监听。

​ vue3中会使用proxy来实现双向绑定，相对于以前的数据劫持来说，这种方法应该说是拦截，外界对于数据的访问都必须通过这层拦截，它直接监听的是对象而非属性，同时对数组的支持度也非常高，返回一个新对象，可以直接操作一个新对象来达到目的，而不用像以前那样遍历对象属性。

九、虚拟dom

​ 为什么要使用虚拟DOM，我们知道前端优化的一个方法就是尽可以减少DOM操作，因为DOM操作慢，而且频繁操作会造成浏览器的回流等，所以把DOM抽象成虚拟DOM，把所有的变动更新完成后一次性映射到真实的DOM树上，而且虚拟的DOM本身是一个对象，操作起来也是更加方便。

​ 虚拟DOM的核心是diff算法，目的就是对比新老树的差异并且更新，它的时间复杂度理论上高达O(N^3)，这是非常耗时间的，不过现在都是同级比较差异，这样来算，它的时间复杂度就是O(n)，就要快很多了

十一、vuex

  • state
    vuex使用单一状态树，即每个应用将仅仅包含一个store实例，但单一状态树和模块化并不冲突。它里面存放数据状态，不可以直接修改里面的数据。
  • mutations
    其定义的方法动态修改store中的数据，store.commit使用
  • getters
    类似vue的计算属性，主要用来过滤一些数据
  • action
    可以将mutations里面的方法变成可异步的处理方式，就是异步操作数据，store.dispath使用

十二、node

​ 好处：处理高并发，事件驱动，轻量，可以用于搭建高性能的web服务器

  1. 它是一个Javascript运行环境

  2. 依赖于Chrome V8引擎进行代码解释

  3. 事件驱动
    事件驱动模型:当服务端收到请求时，就把它关闭 然后处理下一个请求 当第一个请求处理完毕后 就放回处理队列 当达到队列开头 将结果返回给用户 好处：高效 扩展性强 因为服务端一直接受请求 不等待任何读写操作.

    事件循环：查看队列里面是否有队列里面有待处理的 如果有 交给主线程执行。

  4. 非阻塞I/O
    因为js是单线程的，不能像后端语言那样创建多个线程执行程序，那就用了非阻塞式I/O，当线程在执行I/O操作时，不用阻塞等待，只需要监听完毕后回调就可以了，其优点是性能强，同样的服务器，node可以容纳传统后端语言一百倍的用户，缺点也很明显，因为是单线程，线程一旦崩溃，所有程序都会崩溃。

  5. 轻量、可伸缩，适于实时数据交互应用

  6. 单进程，单线程
